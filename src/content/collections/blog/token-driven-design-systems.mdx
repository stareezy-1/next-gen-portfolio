# Building Token-Driven Design Systems

Design tokens are the foundation of a scalable design system. They represent the smallest, most atomic design decisions — colors, spacing, typography, shadows — as named constants that can be consumed across platforms.

## What Are Design Tokens?

A design token is a named value that represents a design decision. Instead of hardcoding `#00ff88` in your CSS, you reference `colors.brand.primary`. This indirection gives you:

- **Consistency**: every component uses the same value
- **Maintainability**: change the token, update everywhere
- **Cross-platform**: the same token works in web CSS, React Native StyleSheet, and native iOS/Android

## Structuring Your Token System

A well-structured token system has three tiers:

1. **Primitive tokens**: raw values (`colors.green[500] = "#00ff88"`)
2. **Semantic tokens**: purpose-driven aliases (`colors.brand = colors.green[500]`)
3. **Component tokens**: component-specific overrides (`button.background = colors.brand`)

## Implementing in TypeScript

```typescript
interface Token<T> {
  value: T;
  description?: string;
}

const colors = {
  brand: { value: "#00ff88" } satisfies Token<string>,
  surface: { value: "#0a0a0f" } satisfies Token<string>,
};
```

## CSS Custom Properties as the Transport Layer

CSS custom properties (variables) are the ideal runtime transport for web tokens:

```css
:root[data-palette="aurora"][data-theme="dark"] {
  --color-brand: #00ff88;
  --color-surface: #0a0a0f;
}
```

Switching themes becomes a single attribute change on `<html>` — zero re-renders, no flash.

## Conclusion

Token-driven design systems pay dividends in consistency, maintainability, and cross-platform reach. The upfront investment in a solid token architecture saves significant time as your product scales.
