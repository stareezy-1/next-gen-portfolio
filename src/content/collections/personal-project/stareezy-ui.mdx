Stareezy UI is a design system built for cross-platform consistency. Lead Architect & Maintainer.

## Packages

- `@stareezy-ui/tokens` — zero-dependency token definitions
- `@stareezy-ui/components` — 31+ cross-platform UI components
- `@stareezy-ui/runtime` — O(1) style registry with web and RN adapters
- `@stareezy-ui/compiler` — Babel/Vite build-time transform for atomic CSS
- `@stareezy-ui/core` — utilities, hooks, platform helpers
- `@stareezy-ui/stylesheet` — atomic CSS sheet management

## AI integration layer

I built the design system to be usable from inside an AI assistant, not just an editor:

- An MCP (Model Context Protocol) server exposing 8 read-only tools (`list_components`, `get_token`, `scaffold_component`, `search_docs`, and more) so an assistant generates code against the real API instead of guessing
- An open-source Claude skills collection (tokens, components, themes, best practices) that teaches assistants the three-file component convention and token-accessor rules
